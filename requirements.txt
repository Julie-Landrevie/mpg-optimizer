# ============================================================
# requirements.txt
# ============================================================
# Ce fichier liste toutes les bibliothèques Python dont le projet a besoin.
#
# Comment l'utiliser ?
#   pip install -r requirements.txt
#
# C'est quoi une bibliothèque ?
#   Un ensemble de fonctions déjà écrites par d'autres développeurs,
#   qu'on peut réutiliser pour ne pas tout recoder from scratch.
#   Exemple : pandas nous évite de coder notre propre système de tableaux.
#
# C'est quoi le >= après le nom ?
#   >= 2.0.0 signifie "version 2.0.0 ou plus récente".
#   On évite de fixer une version exacte (==) pour garder de la flexibilité,
#   mais on fixe un minimum pour s'assurer que les fonctions qu'on utilise existent.
# ============================================================
